Bộ Prompt Handoff Vạn Năng

Khung quản lý ngữ cảnh dự án tối ưu tuyệt đối, áp dụng cho mọi ngôn ngữ và nền tảng

Bước 1
Kết Thúc Session Hiện Tại (Compact & Save Context)
Khi nào cần dùng?

Dùng ở cuối mỗi ca làm việc khi AI báo sắp hết dung lượng chat (Limit), hoặc khi mày chủ động muốn dừng tiến độ để hôm sau làm tiếp ở phòng chat mới.

Trước khi kết thúc ca làm việc này, hãy cập nhật và đồng bộ lại toàn bộ trạng thái dự án vào file cấu trúc định danh (ví dụ: claude.md hoặc tương đương) với các mục sau: 1. TIẾN ĐỘ CHI TIẾT: Liệt kê chính xác những file, hàm, component hoặc API nào ĐÃ HOÀN THÀNH trong phiên chat này. 2. KIẾN TRÚC HIỆN TẠI: Tóm tắt cấu trúc cơ sở dữ liệu, luồng đi của biến số (State) hoặc sơ đồ kết nối hạ tầng tính đến thời điểm hiện tại. 3. QUYẾT ĐỊNH CỐ ĐỊNH: Ghi lại các quyết định quan trọng về mặt UI/UX, logic nghiệp vụ cốt lõi, hoặc quy tắc bảo mật đã thống nhất và lý do tại sao chọn phương án đó. 4. KẾ HOẠCH PHIÊN SAU: Chỉ ra chính xác các task cụ thể, các file cần chỉnh sửa hoặc viết mới ngay khi mở phiên làm việc tiếp theo.
Bước 2
Khởi Động Session Mới (Load Context)
Khi nào cần dùng?

Dùng ở câu lệnh đầu tiên ngay khi mở một ô chat mới. Nhớ đính kèm hoặc dán nội dung file cấu trúc định danh (nhật ký) của dự án vào cùng lệnh này.

Hãy đọc kỹ file nhật ký dự án (claude.md / thông tin context) tao vừa cung cấp và phân tích ngắn gọn: 1. Hiện trạng hệ thống đang dừng chính xác ở vị trí nào và mục tiêu tiếp theo cần xử lý là gì? 2. Có quy tắc bọc thép, ràng buộc kỹ thuật hay kiến trúc bảo mật đặc biệt nào cần tuân thủ nghiêm ngặt để đéo làm hỏng hoặc triệt tiêu logic của các phần code cũ đã chạy ổn định không? YÊU CẦU ĐẦU RA: Khi viết code, bắt buộc phải xuất đầy đủ 100% cấu trúc và mã nguồn hoàn chỉnh qua Artifacts (hoặc khối code block sạch). Tuyệt đối nghiêm cấm viết tắt, nghiêm cấm dùng mã giả (placeholder) dạng "// code xử lý tại đây". Toàn bộ comment giải thích code phải sử dụng TIẾNG VIỆT 100%.
Bước 3
Kích Hoạt Plan Mode Cho Task Phức Tạp (Risk & Logic Analysis)
Khi nào cần dùng?

Dùng trước khi bắt đầu viết một tính năng khó, thuật toán phức tạp, hoặc khi phải liên kết xử lý dữ liệu giữa nhiều bảng/hệ thống với nhau nhằm né lỗi triệt tiêu công năng.

Dựa trên cấu trúc toàn cục của dự án và trạng thái hiện tại, hãy kích hoạt "Plan Mode" để phân tích giải pháp cho tính năng lớn chuẩn bị triển khai: - Hãy bóc tách luồng xử lý dữ liệu (Data Flow) đầu vào và đầu ra của tính năng này. - Chỉ ra các điểm rủi ro kỹ thuật, lỗi bất đồng bộ hoặc xung đột logic có thể xảy ra với hệ thống hiện tại và đề xuất giải pháp xử lý triệt để. - Lập lộ trình triển khai chi tiết theo thứ tự ưu tiên (Task nào là điều kiện cần làm trước, task nào làm sau). CHÚ Ý: Nghiêm cấm viết code ngay lập tức. Hãy xuất bảng kế hoạch phân tích logic này bằng tiếng Việt rõ ràng để tao duyệt trước!
